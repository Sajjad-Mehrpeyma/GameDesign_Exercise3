# Pepsi Man Runner (Unity 3D)

یک بازی رانر سه‌لاین ساده برای یادگیری مفاهیم پایه Unity: حرکت خودکار، جابه‌جایی بین لاین‌ها، موانع تصادفی، امتیاز و Game Over.

## پیش‌نیاز

- Unity: نسخه `6000.2.10f1` (یا نزدیک به آن)

## اجرا

1. پروژه را در Unity باز کنید.
2. صحنه `Assets/Scenes/Main.unity` را باز کنید.
3. دکمه Play را بزنید.

«برای اجرای بازی، صحنه Main را در Unity باز کرده و دکمه Play را بزنید.»

## کنترل‌ها

- `A` = چپ، `D` = راست (یا از کلیدهای جهت‌دار چپ/راست استفاده کنید)

## ویژگی‌ها

- حرکت خودکار بازیکن رو به جلو
- سه لاین ثابت (چپ، وسط، راست)
- موانع تصادفی با فاصله ۳ تا ۷ متر
- تشخیص برخورد با Trigger و Game Over
- امتیاز لحظه‌ای و Best Score ذخیره‌شده با PlayerPrefs
- پنل Game Over و دکمه Restart

## پوشه‌ها

- `Assets/Scripts`: اسکریپت‌های اصلی
- `Assets/Prefabs`: پریفب بازیکن/مانع (پس از ساخت در ادیتور)
- `Assets/Materials`: متریال‌های ساده
- `Docs`: فایل‌های گزارش/مستندات (در صورت وجود)

## بازتاب (حدود 250 کلمه)

در این پروژه یاد گرفتم چگونه با استفاده از `Instantiate` موانع را به‌صورت تصادفی در سه لاین ایجاد کنم و فاصله‌ی تصادفی بین ۳ تا ۷ متر بگذارم تا بازی یکنواخت نباشد. برای ذخیره رکورد از `PlayerPrefs` استفاده شد؛ این کار ساده است و نیاز به دیتابیس ندارد. چالش اصلی مدیریت Game Over بود؛ با فعال‌کردن `Is Trigger` روی Collider موانع و پیاده‌سازی `OnTriggerEnter` در Player، تشخیص برخورد ساده شد. پس از برخورد، با متوقف‌کردن زمان (`Time.timeScale = 0`) و نمایش پنل Game Over، جریان بازی مدیریت می‌شود. همچنین دکمه Restart با `SceneManager.LoadScene` صحنه را ریست می‌کند. در بخش UI یاد گرفتم چطور Text را برای امتیاز لحظه‌ای و Best Score به‌روز کنم و چگونه پنل‌ها را فعال/غیرفعال کنم. مهم بود که نام‌گذاری متغیرها واضح باشد (مثلاً `forwardSpeed` و `laneOffset`) تا کد خوانا بماند. در نهایت فهمیدم داشتن ساختار پوشه‌ی منظم (Scripts، Prefabs، Materials) و کامنت‌های کوتاه فارسی در اسکریپت‌ها به درک سریع‌تر کمک می‌کند. اگر زمان بیشتری داشتم، انیمیشن دویدن و صداگذاری اضافه می‌کردم تا حس بهتری بدهد، اما حتی با مدل‌های ساده، هسته‌ی گیم‌پلی به‌خوبی منتقل شد.
